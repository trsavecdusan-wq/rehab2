# CODEX RULES

## PROJECT
rehab2

## MAIN RULES
- stability first
- no uncontrolled refactors
- no file renames without approval
- no dependency changes without approval
- no UI redesign without approval
- always inspect before editing
- always report changed files
- always report build status
- always preserve existing IDs and logic

## SAFE WORKFLOW
1. inspect
2. explain risk
3. propose patch
4. wait approval
5. edit
6. validate

## FORBIDDEN
- deleting unrelated files
- changing package names
- changing signing config
- changing applicationId
- changing Gradle structure without approval
